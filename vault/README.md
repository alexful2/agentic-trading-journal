# vault/ — the example vault

This is a lightly-anonymized snapshot of a real working vault — the actual files
the skills read and write, with secrets and personal identifiers removed. It's
here so you can see the system running on a real book, not a toy.

> **Not investment advice.** Tickers, theses, and analysis are one investor's
> real notes shown as a software demo — not recommendations, endorsements, or
> forecasts. Replace this folder with your own vault to make the system yours.

## What's here

- `watchlist.md` — the control panel: tiers, price triggers, planned tranches.
- `deep-dives/` — three real dives from the same early-June week (IREN HOLD, CRWV
  HOLD, NBIS ADD — three verdicts, one macro backdrop) + the `_verdicts.md` ledger.
- `companies/IREN/` — a full company dossier (8 SEC files) + the **mosaic**
  research layer (`projects` → `execution-thesis` → `execution-audit`).
- `reports/` — a daily alert, the weekly review, and a pre-earnings plan.
- `notes/` — a per-stock thesis note (the "why I own it" layer).
- `tripwires.md` · `price-triggers.md` · `ipo-calendar.md` · `DASHBOARD.md` — the
  falsification ledger, broader-universe triggers, IPO reminders, and the
  at-a-glance dashboard.
- `library/` — reusable decision frameworks + the C1–C15 empirical-priors catalog
  (see `library/README.md`).
- `templates/` — blank templates for trade decisions and trade-close arcs.

Only one ticker (IREN) ships the *full* dossier + mosaic; other names are
referenced but not all included — that's how a real vault looks (the deep-dive
auto-cleanup keeps only the latest file per ticker).

## Make it yours

Delete the content in `notes/`, `deep-dives/`, `reports/`, `companies/`, and the
control files, then start writing your own theses and positions. The skills read
whatever is in the vault — the moment you start journaling here, it becomes
*your* journal. The `library/` frameworks and priors are a starter set; keep,
edit, or delete them.
