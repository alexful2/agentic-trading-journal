# Macro-Cohort Confirmation Before Single-Name Adds

For high-beta single names, only execute pre-planned dip-tranche adds when the
cohort is also red.

**Proceed:** the broad index is down *and* the target's close peers are also down
hard the same day = macro-driven. This is the liquidity event the tranche plan
was designed for.

**Stand down:** the index is roughly flat while the target name sells off alone =
idiosyncratic. Likely a dilutive raise, an analyst downgrade, or an execution
slip. The dip-buy thesis doesn't apply to company-specific bad news.

**Underlying belief:** idiosyncratic weakness in a thesis-confirmed name is
*information*; cohort weakness in the same name is *liquidity*. Information
changes the thesis. Liquidity creates the entry.

**Cuts both ways:** the rule prevents catching falling knives on company-specific
news, *and* prevents hesitation when macro panic creates a real opportunity. The
common failure is the second one — freezing during an index-wide drawdown and
missing the entry the plan was built for.

The mechanical check: when a tranche level triggers, look at the index and two or
three peers before acting. Red across the board → proceed. Red alone → find out
why first.
